1 OVERVIEW

Axis camera products with firmware version 4.00 and above support dynamic parameters. This new functionality and the improvement in Axis products require a new way of interfacing with them. This is the main reason for the conversion from the Axis Video HTTP API version 1 to version 2.  

This document describes the differences between Axis Video HTTP API version 1 and version 2. 

1.1 Product and firmware versions

The support for the HTTP APIs is heavily dependent on the product and firmware release. For compliance information, please refer to the product's Release Notes.

3 API GROUPS

3.1 General

3.1.1 Add, update, remove and list parameters and their values

The requests in this group provide a new and more consistent way of getting and setting parameters in Axis products with firmware version 4.00 and above. The parameters have three different security levels, which level to use depends on which operation shall be done and which parameter shall be operated on. The security levels for each parameter are specified in the parameter document.

For the sake of backwards compatibility, the getparam.cgi and setparam.cgi requests in the HTTP API version 1.xx  will continue to be supported by these products for some time to come. However, these requests are obsolete and should not be used.

This new cgi group also allows the adding and removal of dynamic parameters.    

The parameter nosync has been removed.

3.2.3 Bitmap

Bitmap support is a new feature in some Axis 4.xx products. The requests in this section are very similar to the JPEG requests.

 

3.2.4 JPEG/MJPG

3.2.4.1 JPEG image request

This request is slightly modified. 

  • The parameter <name> from the HTTP API version 1.xx is no longer available.
  • New valid values for <camera> have been added.

 

3.2.4.2 JPEG image (snapshot) CGI request

This request is slightly modified.

  • New valid values for the parameter resolution have been added.
  • New valid values for the parameter camera have been added.
  • There is a new parameter color.
  • There is a new parameter textstring.
  • There is a new parameter textcolor.
  • There is a new parameter textbackgroundcolor.
  • There is a new parameter textpos.
  • There is a new parameter overlayimage.
  • There is a new parameter overlaypos.
  • There is a new parameter squarepixel.
  • The parameter quad is no longer available.
  • The parameter showlength is no longer available. The Content-Length is always included in the HTTP-header.
  • The parameter timeout is no longer available.

 

3.2.4.3 JPEG image response

The field "Content-Length" is always included, and the field "Content-Auth" can no longer be included.

 

3.2.4.4 JPEG buffer request

This request is slightly modified.

  • The parameter do has a new valid value; remove, and the meaning of the value reset has been changed.
  • The time between prealarm and postalarm images is changed. The images will be produced as fast as possible if no predelay and postdelay have been set. The default delay was previously set to 1000 ms.

 

3.2.4.5 MJPG video request

This request is slightly modified. 

  • The parameter <name> from the HTTP API version 1.xx is no longer available.
  • New valid values for <camera> have been added.

 

3.2.4.6 MJPG video CGI request

This request is slightly modified.

  • New valid values for the parameter resolution have been added.
  • New valid values for the parameter camera have been added.
  • There is a new parameter color.
  • There is a new parameter textstring.
  • There is a new parameter textcolor.
  • There is a new parameter textbackgroundcolor.
  • There is a new parameter textposition.
  • There is a new parameter textpos.
  • There is a new parameter overlayimage.
  • There is a new parameter overlaypos.
  • There is a new parameter squarepixel.
  • There is a new parameter fps.
  • The parameter quad is no longer available.
  • The parameter showlength is no longer available. The Content-Length is always included in the HTTP-header.
  • The parameters req_fps and des_fps are no longer available.
  • The parameter deltatime is no longer available.
  • The parameter timeout is no longer available.

 

3.2.4.7 MJPG video response

The field "Content-Length" is always included, and the field "Content-Auth" can no longer be included.

This is a new feature in some Axis 4.xx products.

 

3.2.5 MPEG-4

MPEG-4 is a new feature in some Axis 4.xx products.

 

3.3 PTZ

The PTZ framework is completely new in 4.xx products. 

  • These products have no PTZ drivers installed from start, a driver must be uploaded and associated with a serial port and then the camera must be connected to the serial port. 
  • To set the PTZ parameters, the ptzparam.cgi is used. Listing the PTZ parameters is done with param.cgi like any other parameters.
  • PTZ control queue is a new feature in 4.xx products.

Modifications made to control PTZ:

  • PTZ preset configuration is moved from ptz.cgi to ptzconfig.cgi due to security reasons.
  • There is a new parameter home.
  • There is a new parameter bartype.
  • There is a new parameter imagerotation.
  • The parameter zoomrel is no longer available.
  • The parameter focusrel is no longer available.
  • The parameter irisrel is no longer available.
  • The meaning of the value to the parameter rfocus has changed.

 

3.4 Motion Detection

The Motion Detection framework is completely new. 

  • A Motion Detection window is created when adding a group of Motion parameters. The parameters are dynamically created and are created, modified, listed and removed with param.cgi. 
  • It is now possible to define exclude windows, i.e. defining that motion in a specific area should be ignored.
  • To get the Motion Detection levels, motiondetect.cgi is used, with ability to define which MD windows are of interest. It is also possible to get the MD levels related to a specific window configuration.

 

3.5 I/O

3.5.1 I/O control

3.5.1.1 Input

This request is slightly modified.

  • There is a new parameter checkactive.

 

3.5.1.2 Output

This request is slightly modified.

  • There is a new parameter checkactive.

 

3.5.2 Virtual I/O control

3.5.2.1 Input

The state of the virtual input control has changed. The virtual input is simulating a real electrical input and can either be activated or deactivated. This means that after having activated the virtual input, it must first be deactivated to be able to activate it again.

 

3.6 Event

The event administration is completely changed and settings are now stored as any other parameter, meaning that param.cgi is used to add, modify and remove events.

 

3.7 Firewall - IP filter

The firewall (IP address filtering) administration is completely changed. The IP filter settings can be made either by changing the parameters using param.cgi or by using the ipfilter.cgi. The ipfilter.cgi offers possibility to verify that you are able to connect to the Axis device when the IP filter is enabled. 

 

3.8 Audio

The Audio requests are the same but now with ability to choose streaming method.

  • There is a new parameter httptype.

param.cgi can be used to get and set the Audio parameters, but for the sake of backwards compatibility, the /axis-cgi/audio/getparam.cgi requests in the HTTP API version 1 will continue to be supported by these products for some time to come.
